## Review

### Blockers
- `src/lib/chat-reconciliation.ts:70` / `src/lib/chat-reconciliation.ts:188-195`: terminal artifact reconciliation only checks whether the persisted snapshot contains **any** terminal artifact tool (`hasTerminalArtifactTool(freshMessages)`). For H2O package turns with multiple artifact tools, a snapshot with one terminal artifact and another still `input-available`/streaming would pass this guard and be applied. That does not fully satisfy “waits for persisted terminal artifact tool states before applying for active artifact turns”; it should require the relevant active artifact tools for the turn to be terminal, not just one terminal artifact somewhere in the message history. The current tests cover one active artifact only (`src/components/chat-interface.test.tsx:399`, `src/components/chat-interface.test.tsx:468`), so this multi-artifact stale-apply path is untested.
